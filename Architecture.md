The system is refactored into a layered architecture centered on a 'Result' pattern for error propagation. 1. Domain Layer: Defines JWTError and its subclasses to provide granular feedback. 2. Data Model: JWTPreview now includes an 'errors' list to consolidate segment-level failures without breaking the return type consistency. 3. Transformation Layer: JWTDecoder is fully decoupled from I/O, utilizing functional purity for decoding segments. 4. Adapter Layer: CLIHandler handles stream lifecycle management, including graceful handling of BrokenPipeError for shell piping.
